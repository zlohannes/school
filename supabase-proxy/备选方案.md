# 国内代理备选方案

由于 Cloudflare 访问受限，提供以下替代方案：

---

## 方案一：Vercel Edge（可能可访问）

官网：https://vercel.com

如果 Vercel 可以访问，部署步骤类似 Cloudflare。

---

## 方案二：直接换国内 BaaS（推荐长期方案）

### LeanCloud（最推荐）
- 官网：https://leancloud.cn
- 国内服务器，速度极快
- 免费额度：每天 3 万次请求，足够论坛使用
- 数据存储、用户系统、实时消息都有

**迁移成本：** 需要 1-2 天重写数据层代码

### Bmob
- 官网：https://www.bmob.cn
- 国内 BaaS 服务
- 免费额度较宽松

### 腾讯云 CloudBase
- 官网：https://cloud.tencent.com/product/tcb
- 国内节点
- 有免费额度

---

## 方案三：静态化 + 增量更新（折中方案）

不改变数据库，但通过 GitHub Actions 定时把数据导出为 JSON 文件：

```
Supabase → GitHub Actions(每小时) → JSON 文件 → GitHub Pages
```

**优点：**
- 用户直接访问 GitHub Pages，速度极快
- 帖子列表秒开
- 实现简单

**缺点：**
- 数据有 1 小时延迟
- 发帖/回复还是需要连 Supabase

---

## 方案四：分区域部署（高级方案）

1. **帖子列表** → 使用静态 JSON（GitHub Pages 托管）
2. **发帖/回复/登录** → 使用 Supabase（功能可用但慢）

这样用户看帖子很快，互动功能慢一点但能接受。

---

## 推荐方案

| 你的情况 | 推荐方案 |
|----------|----------|
| 想快速解决，不改代码 | 方案三：静态化 + 增量更新 |
| 想彻底解决，愿意改代码 | 方案二：迁移到 LeanCloud |
| 想保留 Supabase | 尝试找能访问的代理服务 |
